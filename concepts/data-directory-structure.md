## How It Works

- Each script output goes to `data/runs/$(date +%F)/filename.json`
- `data/latest/status.json` is generated by `tools/generate_status.py` after any pipeline run
- The agent reads `data/latest/status.json` at session start (~2.5KB) instead of reading 5+ files
- `data/organize_data.py` moves files into runs/ and creates latest/ symlinks
- Can be run partially (e.g., only keyword_discovery creates a new run entry)

## Session Startup
The agent reads `data/latest/status.json` via [[seo-geo-master-agent-loop]] at session start:

```python
# One read вЂ” full status
status = json.loads(Path("data/latest/status.json").read_text())
print(status["summary"])
# в†’ "site=FAIR (55/100) | sandbox=LATE_SANDBOX | keywords=149 | gaps:P1=0"
```

## Outdated Detection

`status.json` includes an `outdated` dict that flags data needing refresh:
- `"ranked": "Missing intent field вЂ” needs trend_validator re-run"`
- `"serp": "Old format вЂ” needs refresh for URL classification"`

This drives the agent's autonomous scheduling decisions.
